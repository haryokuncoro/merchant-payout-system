1. Get the application status by running these commands:
  kubectl get pods -n {{ .Release.Namespace }}
  kubectl get svc -n {{ .Release.Namespace }}

2. Wait for all pods to be ready (especially the database and Kafka):
  kubectl wait --for=condition=ready pod -l app.kubernetes.io/name=merchant-payout-system -n {{ .Release.Namespace }} --timeout=300s

3. Access the application:
{{- if .Values.ingress.enabled }}
  The application is exposed via Ingress at: https://{{ (index .Values.ingress.hosts 0).host }}
{{- else }}
  {{- if eq .Values.service.type "LoadBalancer" }}
  Get the external IP:
    kubectl get svc -n {{ .Release.Namespace }} {{ include "merchant-payout-system.fullname" . }}
  Then access at: http://<EXTERNAL-IP>:{{ .Values.service.port }}
  {{- else }}
  Port forward to the service:
    kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "merchant-payout-system.fullname" . }} 8080:{{ .Values.service.port }}
  Then access at: http://localhost:8080
  {{- end }}
{{- end }}

4. View application logs:
  kubectl logs -f -n {{ .Release.Namespace }} deployment/{{ include "merchant-payout-system.fullname" . }}

5. Verify database connection:
  kubectl exec -it -n {{ .Release.Namespace }} deployment/postgresql -- psql -U {{ .Values.postgresql.auth.username }} -d {{ .Values.postgresql.auth.database }} -c "SELECT version();"

6. Verify Kafka is running:
  kubectl exec -it -n {{ .Release.Namespace }} deployment/kafka -- kafka-broker-api-versions --bootstrap-server localhost:9092

7. Connect to services locally:
  # Database
  kubectl port-forward -n {{ .Release.Namespace }} svc/postgresql 5432:5432

  # Kafka
  kubectl port-forward -n {{ .Release.Namespace }} svc/kafka 9092:9092

  # Application
  kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "merchant-payout-system.fullname" . }} 8080:8080

8. Troubleshooting:
  # Check pod status and events
  kubectl describe pod <pod-name> -n {{ .Release.Namespace }}

  # Check resource usage
  kubectl top pods -n {{ .Release.Namespace }}

  # Check PVC status
  kubectl get pvc -n {{ .Release.Namespace }}

9. To uninstall the release:
  helm uninstall {{ .Release.Name }} -n {{ .Release.Namespace }}

For more information, please refer to the README.md in the helm chart directory.
